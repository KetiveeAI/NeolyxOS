🎯 Critical Issues Found

Issue #1: ELF BSS Segment Size Limitation

Problem: The linker loads the desktop ELF, but the BSS segment (uninitialized data) doesn't get enough physical memory allocated for the 32MB static heap.

desktop/include/nxsyscall.h

// Current implementation: 32MB static heap in BSS
static uint8_t heap[32*1024*1024];  // 32MB

desktop/user.ld

SECTIONS {
    . = 0x400000;  /* 4MB mark */
    .text : { *(.text .text.*) }
    .rodata : { *(.rodata .rodata.*) }
    .data : { *(.data .data.*) }
    .bss : { *(.bss .bss.*) *(COMMON) }  /* 32MB heap here! */
}

kernel/src/loader/elf_loader.c

// ELF loader zeros BSS section:
if (phdr->p_memsz > phdr->p_filesz) {
    uint64_t bss_start = load_addr + phdr->p_filesz;
    uint64_t bss_size = phdr->p_memsz - phdr->p_filesz;
    mem_set((void*)bss_start, 0, bss_size);
}
// BUT: physical pages may not be mapped!
Fix: Ensure the ELF loader or paging code maps sufficient pages for the entire p_memsz segment, not just p_filesz. The BSS needs 32MB → 8192 pages.

Issue #2: Back Buffer Allocation Silent Failure

Problem: When nx_alloc() cannot allocate the back buffer (due to insufficient BSS pages), it silently falls back to single-buffer mode.

desktop/shell/desktop_shell.c

g_backbuffer = (uint32_t*)nx_alloc(g_buffer_size);  // ~4MB
if (!g_backbuffer) {
    /* Fall back to single buffering if allocation fails */
    g_backbuffer = (uint32_t*)g_framebuffer;  // CAUSES FLICKER!
}
Fix: Add serial debug output to indicate allocation failure: 
if (!g_backbuffer) {
    nx_debug_print("[DESKTOP] WARNING: Back buffer alloc failed, single-buffering\\n");
    g_backbuffer = (uint32_t*)g_framebuffer;
} else {
    nx_debug_print("[DESKTOP] Double buffering enabled\\n");
}

Issue #3: Unoptimized Buffer Copy

Problem: The flip_buffer() function uses a naive byte-by-byte copy, which is slow on 4MB of data (~1280×800×4 bytes). 
// Current: slow loop copy
for (uint64_t i = 0; i < pixels; i++) {
    dst[i] = src[i];
}
Fix: Use optimized x86_64 REP MOVSQ for 8-byte transfers: 
// Optimized: REP MOVSQ (8 bytes at a time)
static void flip_buffer_fast(void) {
    uint64_t *src = (uint64_t*)g_backbuffer;
    uint64_t *dst = (uint64_t*)g_framebuffer;
    uint64_t count = g_buffer_size / 8;  // 8 bytes per iteration
    __asm__ volatile(
        "rep movsq"
        : "+S"(src), "+D"(dst), "+c"(count)
        :
        : "memory"
    );
}

📚 Research Findings (OSDev Best Practices)

Source: OSDev Wiki, OSDev Forum, Reddit r/osdev

UEFI GOP Framebuffer

  • GOP framebuffer persists after ExitBootServices()
  • Must use uncached memory (PWT+PCD flags or MTRR)
  • No hardware acceleration after boot - software rendering only

Double Buffering Best Practices

  1. Back buffer in system RAM - faster than VRAM for CPU writes
  2. Optimized memcpy - Use REP MOVSQ or SSE/AVX for speed
  3. VSync synchronization - Prevents tearing (optional)
  4. Dirty rectangle tracking - Only copy changed regions

QEMU-Specific Issues

  • BGA (Bochs Graphics Adapter) may flicker on macOS QEMU builds
  • Use -device VGA or -device virtio-gpu for better compatibility
  • Ensure framebuffer address is correctly mapped in page tables
