🥩 TCP/IP Stack Implementation

Source: kernel/src/net/tcpip.c

Socket Layer

NeolyxOS provides a BSD-like socket interface. Sockets are kernel objects (`socket_t`) linked list tracked by file descriptors.

Supported Operations

Function Description
socket_create(type, proto) Allocates new socket (FD > 2).
socket_bind(fd, addr) Binds to local IP/Port.
socket_connect(fd, addr) Initiates TCP 3-way handshake.
socket_listen(fd, backlog) Sets TCP state to LISTEN.
socket_accept(fd, addr) Accepts incoming connection (Stub).
socket_send(fd, buf, len) Transmits data (TCP/UDP).
socket_recv(fd, buf, len) Reads from RX circular buffer.

Protocols

IPv4

  • ARP: Caching enabled (Size: 32 entries). Handles Request/Reply.
  • Routing: Basic subnet check. Uses Gateway for outside subnet.
  • Checksum: Hardware offload assisted (software fallback).

TCP (Transmission Control Protocol)

A state-machine based implementation handling reliability.

  • States: LISTEN, SYN_SENT, SYN_RCVD, ESTABLISHED, CLOSE_WAIT.
  • Window: Fixed 8192 bytes.
  • Buffering: 8KB RX buffer per socket.

UDP (User Datagram Protocol)

  • Stateless datagrams.
  • DNS Integration: Auto-intercepts port 53 responses for DNS resolver.

🔥 Built-in Firewall

The kernel includes a stateful packet filter activated by default.

Default Rules

Rule 0: DENY Port 23  (Telnet)
Rule 1: DENY Port 139 (NetBIOS)
Rule 2: DENY Port 445 (SMB)

DHCP Client

NeolyxOS implements a full DHCP DORA (Discover-Offer-Request-Ack) cycle.

  1. Discover: Broadcasts to 255.255.255.255 (Port 67).
  2. Offer: Parses offered IP/Mask/Router/DNS.
  3. Request: Requests the offered IP.
  4. Ack: Configures interface on confirmation.
