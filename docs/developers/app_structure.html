📦 NeolyxOS App Development Guide

Simple structure for developers to create apps for NeolyxOS.

App Structure (Not a Bundle)

NeolyxOS apps are simple directories with a standard layout. No special packaging required.

Minimal App Structure

MyApp/ ├── MyApp — Executable binary ├── Info.plist — App metadata (required) ├── Resources/ │ ├── icon.png — App icon (256x256) │ ├── en.lproj/ — English strings │ └── assets/ — Images, sounds, etc. └── lib/ — Bundled libraries (optional)

Info.plist (Required)

Simple metadata file describing your app.

{
    "NXAppName": "My App",
    "NXAppId": "com.developer.myapp",
    "NXAppVersion": "1.0.0",
    "NXExecutable": "MyApp",
    "NXIcon": "Resources/icon.png",
    
    "NXCapabilities": [
        "filesystem.user",
        "network.client"
    ],
    
    "NXMinOS": "1.0"
}

Where App Data is Stored

Apps should store data in standard locations, not in the app directory.

Data Type Location Notes
Settings/Preferences ~/Library/Preferences/com.developer.myapp.plist JSON or plist format
App Data (Databases, files) ~/Library/Application Support/MyApp/ User-specific data
Cache (Temporary) ~/Library/Caches/com.developer.myapp/ Can be deleted anytime
Logs ~/Library/Logs/MyApp/ Debug output
App State (Resume) ~/Library/Saved Application State/com.developer.myapp/ Window positions, state
Sandboxed Data ~/Library/Containers/com.developer.myapp/ For sandboxed apps

System Paths

Where to install and find system resources.

Purpose Path
System Apps /Applications/
User Apps ~/Applications/
System PATH Binaries /Library/Bin/
User PATH Binaries ~/.local/bin/
Frameworks /Library/Frameworks/
Python Runtime /Library/Runtimes/Python/
VST Plugins /Library/Audio/Plugins/VST/
Public Shared Data /Public/

Capabilities

Apps declare what they need access to.

Capability Description
filesystem.user Access user's Documents, Pictures, etc.
filesystem.full Full filesystem access (requires admin)
network.client Make outbound network connections
network.server Listen on ports
audio.playback Play audio
audio.record Record from microphone
gpu.render Use GPU for rendering
camera Access camera

Quick Start: Hello World App

1. Create Directory

make dir ~/Applications/HelloWorld

2. Create Info.plist

{
    "NXAppName": "Hello World",
    "NXAppId": "com.example.helloworld",
    "NXAppVersion": "1.0.0",
    "NXExecutable": "hello",
    "NXCapabilities": []
}

3. Compile Your App

# C example
nxcc -o ~/Applications/HelloWorld/hello hello.c

4. Run

~/Applications/HelloWorld/hello

Path Constants (For C Developers)

#include <fs/fs_init.h>

// System paths
FS_PATH_APPLICATIONS  // "/Applications"
FS_PATH_LIBRARY       // "/Library"
FS_PATH_BIN           // "/Library/Bin"
FS_PATH_FRAMEWORKS    // "/Library/Frameworks"
FS_PATH_PYTHON        // "/Library/Runtimes/Python"
FS_PATH_VST           // "/Library/Audio/Plugins/VST"
FS_PATH_PUBLIC        // "/Public"

// Get user directories
fs_get_user_dir(username, FS_DIR_DOCUMENTS, path, sizeof(path));
fs_get_user_dir(username, FS_DIR_DESKTOP, path, sizeof(path));
