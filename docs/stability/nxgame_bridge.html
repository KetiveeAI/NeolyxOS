🎮 NXGame Bridge

Direct hardware access for games, creative apps (After Effects, DaVinci Resolve), and game engines (Unreal, Unity).

NEW in Phase 9 Implemented

Games bypass compositor via EXCLUSIVE MODE 3. When game goes exclusive → NXRender steps out 4. Desktop crash ≠ System crash (kernel keeps running) 5. Game crash releases GPU cleanly, desktop resumes

Features

🖥️ Direct GPU Access

  • Bypass window compositor
  • Direct framebuffer rendering
  • VSync control (on/off)
  • Exclusive fullscreen mode

🔊 Low-Latency Audio

  • ~5ms audio latency
  • Direct audio buffer access
  • Configurable sample rate
  • Priority audio stream

🎮 Raw Input Polling

  • No event queue delay
  • Keyboard state array
  • Mouse delta (FPS games)
  • Gamepad support (via nxusb)

🛡️ Crash Isolation

  • Game crash ≠ system crash
  • GPU released on crash
  • Desktop auto-resumes
  • Resource quotas

Usage Example

// Create game session
int session = syscall(SYS_NXGAME_CREATE, NXGAME_SESSION_GAME, 
                      NXGAME_CAP_GPU | NXGAME_CAP_AUDIO, NULL);

// Start exclusive mode (desktop steps back)
syscall(SYS_NXGAME_START, session, 0, 0);

// Get framebuffer for rendering
nxgame_buffer_t buf;
syscall(SYS_NXGAME_GET_BUFFER, session, &buf, 0);

// Game loop
while (running) {
    // Poll input (no event queue delay)
    nxgame_input_state_t input;
    syscall(SYS_NXGAME_INPUT, session, &input, 0);
    
    // Render to framebuffer
    render_game(buf.virt_addr, buf.width, buf.height);
    
    // Present with VSync
    syscall(SYS_NXGAME_GPU_PRESENT, session, 1, 0);
}

// Cleanup (desktop resumes)
syscall(SYS_NXGAME_DESTROY, session, 0, 0);
