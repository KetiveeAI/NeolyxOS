Animation System

NXRender provides a high-precision animation engine with nanosecond timing, physics-based spring animations, and CSS-compatible bezier easing curves.

How Animation Works in NXRender

The animation system operates on a simple principle:

  1. Create an animation with target values and timing
  2. Start the animation
  3. Update each frame with delta time
  4. Read the interpolated value
  5. Apply to widget properties
// Animation lifecycle nx_keyframe_anim_t* anim = nx_keyframe_anim_create(300); // 300ms nx_keyframe_add(anim, 0.0, 0.0, NX_BEZIER_LINEAR); // Start nx_keyframe_add(anim, 1.0, 100.0, NX_BEZIER_EASE_OUT); // End nx_keyframe_anim_play(anim); // Each frame (60 FPS = 16.67ms per frame) while (nx_keyframe_anim_update(anim, 16.67)) { float value = nx_keyframe_anim_value(anim); widget->bounds.x = value; // Apply to widget redraw(); }

Cubic Bezier Curves

Bezier curves create smooth, natural motion by defining how the animation accelerates and decelerates. A cubic bezier is defined by two control points (P1 and P2):

P(t) = (1-t)³P₀ + 3(1-t)²tP₁ + 3(1-t)t²P₂ + t³P₃

where P₀=(0,0) and P₃=(1,1) are fixed, and P₁=(x1,y1), P₂=(x2,y2) define the curve shape.

Preset Curves

Preset Values (x1, y1, x2, y2) Description
NX_BEZIER_LINEAR 0.0, 0.0, 1.0, 1.0 Constant speed
NX_BEZIER_EASE 0.25, 0.1, 0.25, 1.0 Subtle acceleration/deceleration
NX_BEZIER_EASE_IN 0.42, 0.0, 1.0, 1.0 Starts slow, ends fast
NX_BEZIER_EASE_OUT 0.0, 0.0, 0.58, 1.0 Starts fast, ends slow
NX_BEZIER_EASE_IN_OUT 0.42, 0.0, 0.58, 1.0 Slow start and end
NX_BEZIER_EASE_IN_BACK 0.6, -0.28, 0.735, 0.045 Pulls back before starting
NX_BEZIER_EASE_OUT_BACK 0.175, 0.885, 0.32, 1.275 Overshoots then settles

Spring Physics

Spring animations simulate real-world physics for natural, responsive motion. They're defined by three parameters:

F = -kx - cv
a = F/m
Parameter What it does Effect of increasing
mass Object inertia Slower, heavier feeling
stiffness (k) Spring strength Faster, snappier motion
damping (c) Friction/resistance Less bouncing

Damping Ratio (ζ)

ζ = c / (2 × √(k × m))
ζ Value Behavior Use Case
ζ < 1 (underdamped) Oscillates/bounces Playful UI, notifications
ζ = 1 (critically damped) Fastest without bounce Precise controls
ζ > 1 (overdamped) Slow, no overshoot Heavy objects, modals
// Custom spring nx_spring_adv_t* spring = nx_spring_adv_create( 1.0, // mass 180.0, // stiffness 12.0 // damping (underdamped = bouncy) ); // Or use presets nx_spring_adv_t* wobbly = nx_spring_adv_from_preset(NX_SPRING_WOBBLY); nx_spring_adv_t* stiff = nx_spring_adv_from_preset(NX_SPRING_STIFF); // Animate nx_spring_adv_set_target(spring, 200.0); while (!nx_spring_adv_is_settled(spring)) { nx_spring_adv_update(spring, 16.67); // dt in ms float position = nx_spring_adv_value(spring); // Apply position... }

Animation Timeline

Timelines let you orchestrate multiple animations with precise timing:

nx_timeline_t* timeline = nx_timeline_create(); // Add animations with start times (in ms) nx_timeline_add(timeline, fade_in_anim, 0); // Start immediately nx_timeline_add(timeline, slide_anim, 100); // Start at 100ms nx_timeline_add(timeline, scale_anim, 200); // Start at 200ms nx_timeline_play(timeline); // Update each frame while (nx_timeline_update(timeline, dt_ms)) { redraw(); }

Staggered Animations

Create cascading effects where items animate one after another:

// 10 items, 50ms stagger between each, 300ms per item nx_stagger_t* stagger = nx_stagger_create(10, 50, 300); nx_stagger_play(stagger); // Each frame nx_stagger_update(stagger, dt_ms); // Get value for each item (0.0 to 1.0) for (int i = 0; i < 10; i++) { float progress=nx_stagger_value_at(stagger, i); items[i].opacity=progress; }

Precision Timing

NXRender uses nanosecond-precision timing for frame-perfect animations:

// High-precision time nx_time_ns_t start = nx_time_now(); // ... do work ... nx_time_ns_t elapsed = nx_time_now() - start; float seconds = nx_time_to_seconds(elapsed);

Animation Controller

Manage all active animations centrally:

nx_anim_controller_t* ctrl = nx_anim_controller_create(); ctrl->time_scale = 1.0; // 1.0 = normal, 0.5 = slow-mo, 2.0 = fast // Add animations nx_anim_controller_add(ctrl, my_animation, ANIM_TYPE_KEYFRAME); // Each frame - automatically calculates delta time nx_anim_controller_update(ctrl); // Pause/resume all nx_anim_controller_pause_all(ctrl); nx_anim_controller_resume_all(ctrl);

Best Practices

  • Use springs for interactive elements - They respond naturally to interruption
  • Use bezier for timed animations - When you need precise duration
  • Match animation to content - Light UI = stiff spring, heavy content = slow spring
  • Stagger list animations - Creates visual rhythm and guides attention
  • Keep durations short - 200-400ms feels responsive
