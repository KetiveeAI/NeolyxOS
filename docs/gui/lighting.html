3D Lighting System

NXRender includes a physically-based lighting system that simulates how light actually behaves in the real world. This enables realistic rendering of 3D scenes with proper shadows, reflections, and material properties.

How Light Works in Real Life

When you have a single light source in a dark room, light behaves according to physical laws:

1. Inverse Square Law

Light intensity decreases with the square of the distance. An object twice as far from a light receives only 1/4 the light.

Intensity = 1 / (constant + linear×d + quadratic×d²)

2. Diffuse Reflection (Lambert's Law)

When light hits a rough surface, it scatters equally in all directions. The brightness depends on the angle between the surface and light direction.

Diffuse = max(dot(N, L), 0) × light_color × surface_color

3. Specular Reflection (Blinn-Phong)

Shiny surfaces create highlights. The highlight moves as you change viewing angle.

Specular = pow(max(dot(N, H), 0), shininess) × light_color

where H is the half-vector between light direction and view direction.

4. Fresnel Effect

At grazing angles, surfaces become more reflective. This is why water looks like a mirror when viewed from a low angle.

F = F₀ + (1 - F₀) × (1 - cos θ)⁵

Materials

Different materials interact with light differently:

Property Range Effect
metallic 0.0 - 1.0 0 = dielectric (plastic), 1 = metal
roughness 0.0 - 1.0 0 = mirror, 1 = completely rough
ior 1.0 - 3.0 Index of refraction (glass ~1.5)
ambient_occlusion 0.0 - 1.0 Corners/crevices are darker
// Material presets nx_material_t white_plastic = nx_material_plastic( nx_vec3(1, 1, 1), // White color 0.5 // Medium roughness ); nx_material_t gold = nx_material_metal( nx_vec3(1.0, 0.84, 0), // Gold color 0.3 // Slightly rough ); nx_material_t glass = nx_material_glass(1.5); // IOR of glass

Scene Setup

// Create lighting scene nx_lighting_scene_t* scene = nx_lighting_create(); // Set ambient light (for indirect illumination) nx_lighting_set_ambient(scene, nx_vec3(0.1, 0.1, 0.15), // Slight blue tint 0.1 // Low intensity for dark room ); // Add lights nx_lighting_add_light(scene, bulb); nx_lighting_add_light(scene, sun); // Set camera position (for specular calculation) scene->camera_position = nx_vec3(0, 1, 5);

Calculating Surface Color

// Define surface point to light nx_surface_t surface = { .position = nx_vec3(0, 0, 0), // World position .normal = nx_vec3(0, 1, 0), // Pointing up .view_dir = nx_vec3_normalize( // Camera direction nx_vec3_sub(camera_pos, surface_pos) ), .material = white_plastic }; // Calculate final lit color nx_vec3_t color = nx_lighting_calculate(scene, &surface); // Convert to display color (0-255) uint8_t r = (uint8_t)(color.x * 255); uint8_t g = (uint8_t)(color.y * 255); uint8_t b = (uint8_t)(color.z * 255);

Reflection Calculation

For mirror-like surfaces, you can calculate the reflection direction:

Incident (I) Normal (N) Reflected (R) ╲ │ ╱ ╲ │ ╱ ╲ │ ╱ ╲ │ ╱ ╲ │ ╱ ───────────╲─────┼─────╱─────────── Surface R = I - 2 × dot(I, N) × N
// Calculate reflection direction nx_vec3_t incident = nx_vec3(0.5, -1, 0); // Incoming ray nx_vec3_t normal = nx_vec3(0, 1, 0); // Surface normal nx_vec3_t reflected = nx_calculate_reflection(incident, normal); // Result: (0.5, 1, 0) - bounces up

Single Light in Dark Room

This is a common scenario. Here's exactly what happens:

  1. Ambient is very low - Almost no indirect light bouncing
  2. Objects near the light are bright - Inverse square law
  3. Surfaces facing away are dark - Lambert's law
  4. Specular highlights appear - Only on surfaces facing both light and camera
  5. Shadows are sharp - Single point source creates hard shadows
// Dark room with single light setup nx_lighting_scene_t* scene = nx_lighting_create(); // Very low ambient (near darkness) nx_lighting_set_ambient(scene, nx_vec3(0.02, 0.02, 0.03), 0.05); // Single point light (like a bare bulb) nx_light_t bulb = nx_light_point( nx_vec3(0, 3, 0), // Hanging from ceiling nx_vec3(1.0, 0.95, 0.9), // Warm tungsten color 5.0 // Strong intensity ); bulb.linear = 0.14; // Medium falloff bulb.quadratic = 0.07; // Realistic inverse-square nx_lighting_add_light(scene, bulb); // Now surfaces far from (0, 3, 0) will be very dark // Surfaces directly below will be bright

GPU Integration

For hardware-accelerated lighting, use the GPU interface:

// Initialize GPU nx_gpu_interface_t* gpu = nx_gpu_init(kernel_gpu_context); // Upload lighting data to GPU nx_gpu_upload_scene(gpu, scene); // GPU handles all lighting calculations in parallel nx_gpu_render_lit(gpu, scene);
