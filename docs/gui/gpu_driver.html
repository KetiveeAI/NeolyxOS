GPU Driver Interface

NXRender includes a GPU driver interface for hardware-accelerated rendering. The driver communicates with the NeolyxOS kernel GPU subsystem via command buffers.

Architecture

Command Buffer

NXRender uses a ring buffer to queue GPU commands. This allows the CPU to continue preparing the next frame while the GPU executes the current one.

┌───────────────────────────────────────────────────────────────┐ │ Command Ring Buffer │ ├───────────────────────────────────────────────────────────────┤ │ [CLEAR] [RECT] [RECT] [CIRCLE] [BLIT] [LINE] [PRESENT] [...] │ │ ↑ ↑ │ │ read_idx write_idx │ │ (GPU) (CPU) │ └───────────────────────────────────────────────────────────────┘ CPU writes commands at write_idx GPU reads and executes at read_idx When read_idx catches up to write_idx, GPU is idle
// Command buffer structure typedef struct { nx_gpu_cmd_t commands[4096]; // 4096 commands volatile uint32_t write_index; // CPU writes here volatile uint32_t read_index; // GPU reads here volatile uint32_t fence_value; // Synchronization } nx_gpu_cmd_buffer_t;

Basic Usage

// Initialize GPU driver nx_gpu_driver_t* gpu = nx_gpu_driver_init(); // Enable VSync (prevents tearing) nx_gpu_set_vsync(gpu, true); // Main render loop while (running) { // Begin frame nx_gpu_begin_frame(gpu); // Queue draw commands nx_gpu_cmd_clear(gpu, NX_COLOR_BLACK); nx_gpu_cmd_rect(gpu, (nx_rect_t){100, 100, 200, 150}, NX_COLOR_WHITE, 8); nx_gpu_cmd_circle(gpu, (nx_point_t){400, 300}, 50, NX_COLOR_BLUE); // End frame and present nx_gpu_end_frame(gpu); nx_gpu_present(gpu); // Waits for VSync } // Cleanup nx_gpu_driver_shutdown(gpu);

Draw Commands

Function Description
nx_gpu_cmd_clear(gpu, color) Clear framebuffer
nx_gpu_cmd_rect(gpu, rect, color, radius) Draw rectangle (with optional rounded corners)
nx_gpu_cmd_circle(gpu, center, radius, color) Draw filled circle
nx_gpu_cmd_line(gpu, p1, p2, color, width) Draw line
nx_gpu_cmd_blit(gpu, texture, dst) Draw texture
nx_gpu_cmd_mesh(gpu, verts, indices, count) Draw 3D mesh

Texture Management

// Create texture nx_gpu_texture_t tex = nx_gpu_create_texture(gpu, 256, 256, NX_TEX_RGBA8); // Upload pixel data uint32_t* pixels = malloc(256 * 256 * 4); // ... fill pixels ... nx_gpu_upload_texture(gpu, tex, pixels); free(pixels); // Use texture nx_gpu_cmd_blit(gpu, tex, (nx_rect_t){0, 0, 256, 256}); // Cleanup nx_gpu_destroy_texture(gpu, tex);

Texture Formats

NX_TEX_BGRA8 4 Windows-style byte order
NX_TEX_RGB8 3 24-bit color, no alpha

Buffer Management

// Create vertex buffer nx_gpu_buffer_t vbo = nx_gpu_create_buffer(gpu, sizeof(nx_vertex_t) * 1000, // Size 0 // Usage flags ); // Map for CPU access nx_vertex_t* vertices = nx_gpu_map_buffer(gpu, vbo); // Fill vertices vertices[0] = (nx_vertex_t){ .position = {0, 0, 0}, .normal = {0, 1, 0}, .texcoord = {0, 0}, .color = 0xFFFFFFFF }; // ... nx_gpu_unmap_buffer(gpu, vbo); // Or upload directly nx_gpu_upload_buffer(gpu, vbo, my_vertices, sizeof(my_vertices));

VSync and Frame Timing

// Enable VSync (60 FPS cap, no tearing) nx_gpu_set_vsync(gpu, true); // Get frame timing uint64_t frame_time_us = nx_gpu_get_frame_time_us(gpu); uint32_t fps = nx_gpu_get_fps(gpu); printf("Frame time: %lu us, FPS: %u\n", frame_time_us, fps);

VSync enabled: GPU waits for monitor refresh (typically 60 Hz), prevents screen tearing but adds latency.

VSync disabled: Maximum frame rate, may cause tearing artifacts.

Synchronization

Use fences to synchronize CPU and GPU:

// Submit commands and wait for completion nx_gpu_submit(gpu); nx_gpu_wait_idle(gpu); // Blocks until GPU finishes // Now safe to read back results or reuse resources

Kernel Integration

When integrating with the NeolyxOS kernel GPU driver:

  1. Open /dev/nxgpu device
  2. MMAP command buffer memory
  3. MMAP VRAM for textures
  4. Use IOCTL for synchronization
// Kernel integration (future) gpu->driver_fd = open("/dev/nxgpu", O_RDWR); gpu->cmd_buffer = mmap(NULL, cmd_size, PROT_READ | PROT_WRITE, MAP_SHARED, gpu->driver_fd, CMD_BUFFER_OFFSET); gpu->vram_base = mmap(NULL, vram_size, PROT_READ | PROT_WRITE, MAP_SHARED, gpu->driver_fd, VRAM_OFFSET);
