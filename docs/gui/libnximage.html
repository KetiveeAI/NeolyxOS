Overview

libnximage is a pure userspace library for loading images. No kernel involvement required - it uses existing VFS syscalls (open/read/close) and nx_malloc for memory allocation.

Location desktop/lib/nximage/
Output libnximage.a (~25KB static library)
Formats PNG (full), NXI (vector icons), SVG/AVIF (stub)
Dependencies None (pure C, zero external libs)

Source Files

nxpng.c ~435 lines

PNG decoder with inline DEFLATE decompressor

  • Huffman table building and decoding
  • ZLIB header parsing
  • Filter reconstruction (None/Sub/Up/Average/Paeth)
  • RGB, RGBA, Grayscale color type support
  • 8-bit depth only (no 16-bit or palette)

nxnxi.c ~330 lines

NXI vector icon parser and rasterizer

  • INI-style format parsing
  • Circle and rectangle primitives
  • Solid color fills
  • Theme color support (future)
  • Resolution-independent rendering

nximage.c ~195 lines

Main loader dispatcher

  • Magic byte detection (PNG, NXI, JPEG, etc.)
  • File I/O wrappers using VFS syscalls
  • Error code propagation
  • Format routing to appropriate decoder

nxavif_stub.c Stub

AVIF placeholder (returns NXI_ERR_UNSUPPORTED)

Future: Port dav1d or libgav1 for AV1-based image support

API Reference

Header: nximage.h

typedef struct {
    uint32_t width;
    uint32_t height;
    uint32_t bpp;        // Bytes per pixel (always 4 for RGBA)
    uint8_t *pixels;     // RGBA32 pixel data
} nximage_t;

// Load image from filesystem path
int nxi_load(const char *path, nximage_t *out);

// Free image pixel buffer
void nxi_free(nximage_t *img);

// Error codes
#define NXI_OK              0
#define NXI_ERR_FILE       -1   // File not found / read error
#define NXI_ERR_FORMAT     -2   // Unknown or corrupt format
#define NXI_ERR_MEMORY     -3   // Allocation failed
#define NXI_ERR_UNSUPPORTED -4  // Format not implemented

Usage Example

#include "nximage.h"

void load_icon(void) {
    nximage_t img;
    int err = nxi_load("/System/Icons/app.png", &img);
    
    if (err == NXI_OK) {
        // img.pixels = RGBA32 buffer (img.width * img.height * 4 bytes)
        // img.width, img.height = dimensions
        // img.bpp = 4 (bytes per pixel)
        
        blit_to_framebuffer(img.pixels, img.width, img.height);
        nxi_free(&img);
    } else {
        // Handle error
        debug_print("Image load failed: %d\n", err);
    }
}

NXI Format Specification

NXI is a simple text-based vector icon format for NeolyxOS:

# NeolyxOS NXI v1.0
[icon]
size = 64

[shape.background]
type = circle
cx = 32
cy = 32
r = 30
fill = #3B82F6

[shape.symbol]
type = rect
x = 20
y = 24
w = 24
h = 16
fill = #FFFFFF

Benefits: Human-readable, no binary parsing, easy to generate, theme-aware colors.

cd desktop/lib/nximage
make
# Output: libnximage.a

make
# Links libnximage.a automatically

gcc myapp.c -o myapp -Idesktop/lib/nximage -Ldesktop/lib/nximage -lnximage
