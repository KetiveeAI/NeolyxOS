🖥️ NeolyxOS Desktop Environment

Status: ✅ Running in Userspace (Ring 3)

The NeolyxOS Desktop acts as the primary user interface, functioning as a standalone ELF binary loaded by the kernel into userspace.

Architecture

The desktop shell (desktop_shell.c) coordinates several subsystems:

Component Description Status
Window Manager Handles window creation, movement, stacking, and rendering. Active
Dock Fast application launcher with hover effects and animations. Active
App Drawer Grid view of all installed applications. Init Only
Compositor Combines windows and background into the final framebuffer image. Active

System Calls Used

The desktop interacts with the kernel strictly via syscalls:

  • SYS_FB_MAP: Maps the framebuffer to userspace memory (0xC0000000).
  • SYS_INPUT_POLL: Retrieves mouse and keyboard events.
  • SYS_YIELD / SYS_SLEEP: Manages CPU usage.

Memory Layout

The desktop ELF is loaded at 0x400000. Due to large static buffers (BSS), the kernel maps 2MB of memory for code/data and 512KB for stack.

Initialization Sequence

  1. Entry Point: _start (crt0.S) calls main().
  2. FB Init: Determines resolution and attempts malloc for backbuffer.
  3. WM Init: Sets up window state arrays and default opacity.
  4. Dock Init: Loads appearance settings (icons, size).
  5. Event Loop: Enters desktop_run() to poll inputs and redraw.
