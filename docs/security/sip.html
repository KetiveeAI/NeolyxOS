🔐 System Integrity Protection

Protects critical system files from unauthorized modification by malware or accidental changes.

NEW in Phase 9 Implemented

SVG Icon Test:



⚠️ Important: SIP can only be disabled in Recovery Mode. This prevents malware from disabling protection during normal operation.

Protected Paths

Path Protection Description
/System/ Read-Only + No-Delete Core system files
/Kernel/ Read-Only + No-Delete Kernel and drivers
/Boot/ Read-Only + No-Delete Bootloader files
/Library/ System + Admin System libraries
/bin/ Read-Only Essential binaries
/Applications/ Admin Only User apps (admin to install)
/Users/ No Protection User data
/tmp/ No Protection Temporary files

Protection Flags

Flag Value Effect
SIP_PROT_NONE 0x00 No protection
SIP_PROT_READ_ONLY 0x01 Cannot write or delete
SIP_PROT_NO_DELETE 0x02 Can write but not delete
SIP_PROT_SYSTEM 0x04 Only system processes
SIP_PROT_ADMIN 0x08 Requires admin privilege
SIP_PROT_IMMUTABLE 0x10 Can never be modified

Violation Logging

When a protected path is accessed for modification, SIP logs the violation:

[SIP] *** VIOLATION #1 ***
[SIP]   Path: /System/core/kernel.bin
[SIP]   Operation: write
[SIP]   PID: 42
[SIP]   Protection: 0x03

Disabling SIP

Recovery Mode Only: To disable SIP, boot into Recovery Mode and run: 
sip disable
SIP will be re-enabled on next normal boot.

API

// Check if SIP is enabled
sip_status_t sip_is_enabled(void);

// Check if path is writable
int sip_check_writable(const char *path);

// Check path operation (returns 0 if allowed, -EACCES if denied)
int sip_check_path(const char *path, int operation);

// Get protection level for path
uint8_t sip_get_protection(const char *path);
