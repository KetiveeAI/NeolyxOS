🔍 Security Scanner

A proactive threat detection system that verifies file sources, scans downloads, and protects the system from malicious software.

NEW in Phase 16 Implemented

Scan Workflow

flowchart TD
    A[File Access Request] --> B{Source Detection}
    
    B -- /System/ --> C[Trusted (System)]
    B -- /AppStore/ --> D[Verified (Store)]
    B -- /Downloads/ --> E[Untrusted (Web)]
    B -- /USB/ --> F[Unknown (External)]
    
    C --> G[Allow Execution]
    D --> G
    
    E --> H{Scan Required}
    F --> I[Quarantine]
    
    H -- Clean --> J[Show Warning]
    H -- Infection --> K[Block & Log]
    
    J -- User Approves --> G
    J -- User Denies --> L[Abort]
    
    I --> M[Sandboxed Analysis]

Features

🛡️ Protection Capabilities

  • Source Verification: Automatically detects if a file is from System, AppStore, Web, or External media.
  • Download Protection: Forces scanning for all files downloaded from the internet.
  • Quarantine: Moves suspicious or external files to a safe isolation area (`/System/Quarantine`).
  • Signature Matching: Scans files against a database of known threat signatures (SHA-256).
  • Desktop Alerts: Integrated callback system to show native desktop warnings for risky actions.

API Reference

Function Description
scanner_scan_file(path) Scan a specific file for threats. Returns scan result.
scanner_detect_source(path) Identify file origin (System, AppStore, Download, etc).
scanner_quarantine(path) Move a file to the secure quarantine area.
scanner_set_warning_callback(cb) Register a function to receive desktop alert events.
scanner_verify_signature(path, hash) Cryptographically verify file integrity against expected hash.

Source Types

Type ID Policy
FILE_SOURCE_SYSTEM 0 Trusted - No scanning required
FILE_SOURCE_APPSTORE 1 Verified - Signed by Neolyx
FILE_SOURCE_THIRDPARTY 2 Warning - Ask user confirmation
FILE_SOURCE_DOWNLOADED 3 Scan Required - Must scan before open
FILE_SOURCE_EXTERNAL 4 Quarantine - Sandboxed execution
