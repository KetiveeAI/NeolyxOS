Implementation Details

The firewall is implemented in kernel/src/security/firewall.c and provides a robust filtering mechanism for all network traffic.

Key Features

  • Rule-Based Filtering: Supports up to 256 active rules.
  • Direction Control: Separate chains for INPUT, OUTPUT, and FORWARD.
  • Protocol Filtering: Filter by TCP, UDP, ICMP, or ANY.
  • Granular Control: Match source/destination IP ranges and port ranges.
  • App-Level Control: Restrict network access per application process.
  • Telemetry: Tracks packet match counts and total bytes per rule.

Rule Structure

typedef struct {
    uint32_t id;
    uint8_t  direction;     /* INPUT/OUTPUT/FORWARD */
    uint8_t  action;        /* ALLOW/DENY/LOG */
    uint8_t  protocol;      /* TCP/UDP/ICMP/ANY */
    
    uint32_t src_ip;        /* Source IP */
    uint32_t src_mask;      /* Netmask */
    uint32_t dst_ip;        /* Destination IP */
    uint32_t dst_mask;      
    
    uint16_t src_port_min;  /* Port ranges */
    uint16_t src_port_max;
    uint16_t dst_port_min;
    uint16_t dst_port_max;
} firewall_rule_t;

API Reference

Function Description
firewall_init() Initialize the firewall subsystem.
firewall_enable() Enable packet filtering globally.
firewall_add_rule(rule) Add a new filtering rule. Returns Rule ID.
firewall_check_packet(pkt) Core logic called by network stack to verdict a packet.
firewall_get_stats(stats) Retrieve filtered packet counters and byte counts.
firewall_register_app(pid, flags) Register app with network permissions (IN/OUT/BLOCKED).
firewall_app_allow_port(pid, port) Allow specific outgoing port for an app.

Performance

The firewall is designed for low latency. Rules are linear-scanned array for simplicity (O(N)), which is efficient for typical rule counts (< 100). Future optimizations may include hash maps for connection tracking.
