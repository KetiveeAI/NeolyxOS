🦾 ARM64 (AArch64) Architecture

NEW Added in Phase 11

Overview

NeolyxOS now supports ARM64/AArch64 architecture, enabling the OS to run on:

  • ARM Cortex-A53/A72/A76+ processors
  • Apple Silicon (M1/M2/M3)
  • Raspberry Pi 4/5
  • ARM servers (Ampere, Graviton)
  • QEMU virt machine

Boot Sequence

Firmware (UEFI/U-Boot)
    ↓
EL3 (Secure Monitor) ──→ If present, drop to EL2
    ↓
EL2 (Hypervisor) ──────→ Configure HCR_EL2, drop to EL1
    ↓
EL1 (Kernel) ──────────→ _start entry point
    ↓
at_el1:
    • Set up stack (16KB)
    • Clear BSS
    • Set VBAR_EL1 (exception vectors)
    • Initialize MMU
    ↓
kernel_main_arm64()

Page Tables (4KB Granule)

ARM64 uses 4-level page tables for 48-bit virtual addresses:

Virtual Address (48-bit):
┌────────┬────────┬────────┬────────┬──────────┐
│ L0 idx │ L1 idx │ L2 idx │ L3 idx │  Offset  │
│ [47:39]│ [38:30]│ [29:21]│ [20:12]│  [11:0]  │
└────────┴────────┴────────┴────────┴──────────┘
   9 bits   9 bits   9 bits   9 bits   12 bits

L0 → 512GB regions
L1 → 1GB regions  
L2 → 2MB blocks (or table)
L3 → 4KB pages

Memory Attributes (MAIR)

Index Attribute Usage
0 Device-nGnRnE MMIO (GIC, UART)
1 Normal NC Non-cacheable
2 Normal WT Write-through
3 Normal WB Write-back (RAM)

Exception Vectors

16 exception vectors at VBAR_EL1:

Offset    Exception         Source
0x000     Synchronous       Current EL, SP_EL0
0x080     IRQ               Current EL, SP_EL0
0x100     FIQ               Current EL, SP_EL0
0x180     SError            Current EL, SP_EL0
0x200     Synchronous       Current EL, SP_ELx
0x280     IRQ               Current EL, SP_ELx
0x300     FIQ               Current EL, SP_ELx
0x380     SError            Current EL, SP_ELx
0x400     Synchronous       Lower EL, AArch64
0x480     IRQ               Lower EL, AArch64
0x500     FIQ               Lower EL, AArch64
0x580     SError            Lower EL, AArch64

GIC (Generic Interrupt Controller)

NeolyxOS supports GICv2 and GICv3:

Component QEMU virt Address Purpose
Distributor 0x08000000 Interrupt routing
CPU Interface 0x08010000 Per-CPU interface
Redistributor 0x080A0000 GICv3 per-CPU

Device Tree (FDT)

ARM platforms use Device Tree to describe hardware:

fdt_info_t info;
fdt_parse(dtb_addr, &info);

// Memory regions
for (int i = 0; i < info.mem_region_count; i++) {
    uint64_t base = info.mem_regions[i].base;
    uint64_t size = info.mem_regions[i].size;
}

// UART
uint64_t uart = info.uart_base;  // 0x09000000

// GIC
uint64_t gic = info.gic_dist_base;  // 0x08000000
