📦 Application Architecture

App bundles, installation paths, and permissions.

Application Locations

Path Purpose Who Can Install
/Applications/ System-wide apps Admin users
/Users/{user}/Applications/ Per-user apps Current user
/System/Applications/ Core OS apps OS updates only

App Bundle Structure (.nxapp)

MyApp.nxapp/ ├── Contents/ │ ├── Info.plist — App metadata │ ├── Binaries/ — Executable binaries │ │ └── MyApp — Main executable │ ├── Resources/ — Assets, icons, localization │ │ ├── AppIcon.png │ │ └── en.lproj/ │ ├── Frameworks/ — Bundled frameworks │ └── Plugins/ — Optional plugins └── Signature — Code signature

Info.plist (App Manifest)

{
    "NXBundleIdentifier": "com.example.myapp",
    "NXBundleName": "MyApp",
    "NXBundleVersion": "1.0.0",
    "NXExecutable": "MyApp",
    "NXMinimumOSVersion": "1.0",
    
    "NXCapabilities": [
        "network",
        "filesystem.user",
        "camera"
    ],
    
    "NXURLSchemes": ["myapp"],
    "NXFileTypes": [".myf", ".example"]
}

App Capabilities

Capability Description Requires Prompt
network Internet access No
filesystem.user Read/write user files Yes
filesystem.full Full disk access Yes (Admin)
camera Camera access Yes
microphone Microphone access Yes
location Location services Yes
nxgame.exclusive NXGame exclusive mode No

Installing Applications

# Drag .nxapp to /Applications (GUI)

# Command line install (system-wide)
sudo nxinstall MyApp.nxapp

# User-only install
nxinstall --user MyApp.nxapp
→ Installed to /Users/alice/Applications/

# Run app
open /Applications/MyApp.nxapp
# or
MyApp  # if in PATH

App Sandboxing

All apps run in a sandbox by default:

  • Container: ~/Library/Containers/{bundle-id}/
  • Temp files: Inside container only
  • User files: Only with filesystem.user capability
  • Network: Only with network capability
