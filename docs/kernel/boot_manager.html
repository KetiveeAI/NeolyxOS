🖥️ NeolyxOS Boot Manager Guide

Graphical boot menu system and kernel architecture

Overview

NeolyxOS uses a two-stage boot system:

  1. UEFI Bootloader (BOOTX64.EFI) - Loads kernel.bin into memory
  2. Boot Manager (minimal_main.c) - Shows graphical menu
  3. Full Kernel (main.c) - Initializes all subsystems

Source Files

kernel/minimal_main.c
Boot Manager UI (1400+ lines) - Provides kernel_main() entry point
kernel/src/core/main.c
Full Kernel - Provides nx_kernel_start() for OS initialization
kernel/src/arch/x86_64/startup.S
64-bit assembly startup - Calls kernel_main()
kernel/include/ui/disk_icons.h
16x16 bitmap icons for boot menu
boot/main.c
UEFI Bootloader - Loads kernel.bin from EFI partition

Boot Architecture

┌─────────────────────────────────────────────────────────┐
│  UEFI Firmware                                          │
└───────────────────────┬─────────────────────────────────┘
                        ▼
┌─────────────────────────────────────────────────────────┐
│  BOOTX64.EFI (boot/main.c)                              │
│  • Initialize graphics (GOP)                            │
│  • Load kernel.bin to 0x100000                          │
│  • Pass boot_info pointer in RDI                        │
│  • Exit boot services                                   │
│  • Jump to kernel entry                                 │
└───────────────────────┬─────────────────────────────────┘
                        ▼
┌─────────────────────────────────────────────────────────┐
│  startup.S                                              │
│  • Set up stack                                         │
│  • Clear BSS section                                    │
│  • Call kernel_main(boot_info)                          │
└───────────────────────┬─────────────────────────────────┘
                        ▼
┌─────────────────────────────────────────────────────────┐
│  kernel_main() - Boot Manager (minimal_main.c)          │
│  • Initialize framebuffer                               │
│  • Initialize mouse                                     │
│  • Show graphical boot menu                             │
│  • Handle user input                                    │
└───────────────────────┬─────────────────────────────────┘
         User selects   │  "Boot NeolyxOS"
                        ▼
┌─────────────────────────────────────────────────────────┐
│  nx_kernel_start() - Full Kernel (main.c)               │
│  • Initialize video, GDT, IDT, paging                   │
│  • Initialize heap, drivers, filesystem                 │
│  • Start process manager                                │
│  • Enter shell/desktop                                  │
└─────────────────────────────────────────────────────────┘
Important: Do not rename kernel_main() or nx_kernel_start(). These are called by startup.S and the Boot Manager respectively.

Building

# Build kernel with Boot Manager
cd neolyx-os/kernel
make clean && make

# Update boot image
sudo mount -o loop neolyx.img /tmp/neolyx_mount
sudo cp kernel.bin /tmp/neolyx_mount/EFI/BOOT/kernel.bin
sudo umount /tmp/neolyx_mount

# Test in QEMU
qemu-system-x86_64 -bios /usr/share/ovmf/OVMF.fd \
    -drive file=neolyx.img,format=raw -m 512M -serial stdio
