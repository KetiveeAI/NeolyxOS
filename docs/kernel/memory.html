Physical Memory Manager (PMM) Complete

Design

  • Bitmap allocator (1 bit per 4KB page)
  • Parses UEFI memory map at boot
  • Tracks free, used, reserved pages
  • Zeros pages on allocation (security)
  • Double-free detection

API

// Allocate physical pages
uint64_t pmm_alloc_page(void);
uint64_t pmm_alloc_pages(size_t count);

// Free physical pages
void pmm_free_page(uint64_t addr);
void pmm_free_pages(uint64_t addr, size_t count);

// Reserve region (e.g., MMIO)
void pmm_reserve_region(uint64_t start, uint64_t size);

// Statistics
void pmm_get_stats(pmm_stats_t *stats);
int pmm_is_initialized(void);

Virtual Memory Manager (VMM) Complete

Features

  • 4-level page tables (PML4 → PDP → PD → PT)
  • Higher-half kernel at 0xFFFF800000000000
  • Identity mapping first 4GB (2MB huge pages)
  • PMM integration for dynamic page table allocation
  • Address space management for processes/VMs
  • TLB management (per-page and full flush)

Page Flags

Flag Bit Description
PAGE_PRESENT 0 Page is present in memory
PAGE_WRITABLE 1 Page is writable
PAGE_USER 2 Page accessible from Ring 3
PAGE_HUGE 7 2MB huge page
PAGE_GLOBAL 8 Global (not flushed on CR3 switch)
PAGE_NO_EXECUTE 63 NX bit (no execute)

VMM API

// Map virtual to physical

// Unmap virtual address
void vmm_unmap(uint64_t virt);

uint64_t vmm_get_physical(uint64_t virt);

// Allocate virtual pages (with backing physical)

void vmm_free_pages(uint64_t virt, size_t pages);

// TLB control
void vmm_flush_tlb(uint64_t virt);
void vmm_flush_tlb_all(void);

Memory Layout

0x0000000000000000 - 0x00007FFFFFFFFFFF  User space (128TB)
0xFFFF800000000000 - 0xFFFF8FFFFFFFFFFF  Kernel base (identity)
0xFFFF900000000000 - 0xFFFFFFFFFFFFFFFF  Kernel heap/dynamic

Usage Example

// In kernel_main after PMM init:
vmm_init();

// Allocate 16 pages of virtual memory
uint64_t vaddr = vmm_alloc_pages(16);

// Create process address space
uint64_t proc_space = vmm_create_address_space();
vmm_map_in_space(proc_space, 0x400000, phys_code, PAGE_PRESENT | PAGE_USER);

// Switch to process
vmm_switch_address_space(proc_space);
