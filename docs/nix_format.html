🎨 NIX Icon Format Specification

Custom icon format for NeolyxOS with animation support and compression.

Overview

The .nix format provides efficient icon storage with multiple resolutions, animation, and LZ4 compression.

File Structure

Magic Header (8 bytes)

Offset: 0x00
Value:  NIXICON\x01
Purpose: File identification

Header Structure (18 bytes)

Offset 0x09:
  Width:          u32 (little-endian)
  Height:         u32 (little-endian)
  Color Depth:    u8
  Frame Count:    u32
  Animation Delay: u32 (milliseconds)
  Compression:    u8 (0=none, 1=LZ4)

Color Depths

Value Format Bytes/Pixel
8 Grayscale 1
16 RGB 5-6-5 2
24 RGB 8-8-8 3
32 RGBA 8-8-8-8 4

Example: 64x64 RGBA32

0000: 4E 49 58 49 43 4F 4E 01  # NIXICON\x01
0008: 01                        # Version 1
0009: 40 00 00 00              # Width: 64
000D: 40 00 00 00              # Height: 64
0011: 20                        # 32-bit color
0012: 01 00 00 00              # 1 frame
0016: 00 00 00 00              # No animation
001A: 00                        # No compression
001B: [16,384 bytes RGBA data] # 64×64×4

Performance

Size Load Time File Size
16×16 < 1ms ~1KB
64×64 < 5ms ~16KB
128×128 < 20ms ~64KB

Tools

# Convert SVG to NIX
icon-converter input.svg output.nix --size 64x64

# Animated
icon-converter animation.gif output.nix --animation-delay 100

# Validate
nix-validator icon.nix

API Usage

// Load icon
let icon = NixIcon::read(&mut file)?;

// Access frames
for frame in icon.get_frames() {
    // Process frame
}

// Check animation
if icon.is_animated() {
    let delay = icon.get_animation_delay();
}

System Integration

  • MIME Type: image/x-nixicon
  • Extension: .nix
  • Used for: Desktop icons, file manager, app launcher, system tray
